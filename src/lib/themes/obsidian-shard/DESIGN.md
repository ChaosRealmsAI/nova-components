## 1. 设计故事 (Design Story)

> 用 2-3 段话描述这个主题的灵感来源和情感表达

**灵感来源**：
灵感源自火山黑曜石的锋利边缘与未来主义的赛博水晶界面。结合了地质结构的冷峻几何美感与高科技界面的荧光反馈。它不是陈旧的复古，而是锋利、精确、高对比度的未来工业美学。

**情感表达**：
用户应感受到极致的冷静、精确与高效。每一次交互都像切割水晶一样干脆利落，没有拖泥带水的过渡。高光的反射与深邃的黑暗背景形成强烈的视觉冲击，传达出专业与硬核的科技感。

**目标用户场景**：
适合数据密集型的仪表盘、开发者工具、加密货币交易终端、高精度的配置界面或游戏化的系统设置面板。任何需要用户保持高度专注和精确操作的场景。

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | 直角 (0px) 或极小圆角 (1px-2px) | 强调锐利、精确和工业感，模拟黑曜石的断裂面。 |
| **是否使用切角/异形** | 使用切角 (Clip-path) | 在关键交互元素（如按钮、卡片角落）使用 45 度切角，增强科技感。 |

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | 亮度瞬间提升，边缘发光 | 模拟光线照射在光滑晶体表面的反射。 |
| **Active 效果** | 瞬间下沉 (Scale 0.98) | 物理按键的扎实反馈。 |
| **过渡时长** | 快速 (150ms) | 强调效率与响应速度，拒绝拖沓。 |

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | 无衬线字体 (Sans-serif)，等宽字体 (Monospace) 用于数据 | 清晰、现代、易读。 |
| **大小写** | 标题与标签全大写 (Uppercase) | 增强结构感与秩序感，类似工业铭牌。 |
| **字距** | 标题字距微宽 (Tracking-wide) | 提升全大写排版的呼吸感。 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | 无弥散阴影，使用高光边框 (Glow Border) | 在黑暗背景下，光影比阴影更能区分层级。 |
| **边框风格** | 细锐边框 (1px)，Focus 时发光 | 精确界定区域，发光模拟能量流动。 |

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| Button | 0px, 切角 | 1px solid | 无/Glow | 亮度+Glow | Scale 0.98 |
| Input | 0px | 1px solid (Bottom only?) | 无 | Border color | - |
| Card | 2px | 1px solid | 无 | - | - |
| Badge | 0px | 1px solid | 无 | - | - |
| Dialog | 4px | 1px solid | Drop Shadow Hard | - | - |
| Dropdown | 0px | 1px solid | Drop Shadow Hard | Item highlight | - |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=primary, secondary=secondary, ghost=transparent │
│   │ Input        │ bg-surface-1, border-border, focus 时才用 primary       │
│   │ Card         │ bg-surface-1（容器用中性色，不要用 primary）             │
│   │ Dialog       │ bg-surface-1 或 bg-background                           │
│   │ Dropdown     │ bg-surface-2（悬浮层）                                   │
│   │ Table        │ bg-transparent 或 bg-muted（斑马纹）                     │
│   │ Tabs         │ 激活项才用 primary，未激活用 muted                        │
│   │ Badge        │ 根据语义：default=primary, destructive=红色              │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - Primary 只用于主要 CTA 和激活状态                                      │
│   │ - 容器类组件用 surface/background                                       │
│   │ - 表单组件用 surface + border，focus 时才用 primary                     │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `#00f0ff` (Cyan) | 主要操作、强调 | 未来、能量、精确 |
| `--secondary` | `#ffffff` (White) | 次要操作 | 纯净、高对比 |
| `--accent` | `#ff003c` (Neon Red) | 点缀、吸引注意 | 警示、关键点 |

### 5.2 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `#0a0a0a` | 最底层，深邃黑 |
| `--surface-1` | `#161616` | 卡片、面板 |
| `--surface-2` | `#202020` | 下拉菜单、悬浮 |
| `--surface-3` | `#2a2a2a` | 最高层级 |

### 5.3 色彩关系

**深邃基底，荧光点缀**：整个主题以黑色和极深灰色为基调，营造沉浸感。通过高饱和度的青色（Cyan）作为主色调，模拟激光或电子流，形成极高的视觉对比。白色用于次要强调，确保信息在黑暗中清晰可见。红色用于关键警示或特殊状态。整体色彩关系强调"光"在"暗"中的穿透力。

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了设计语言中定义的圆角/切角？
- [ ] **字体**：是否使用了设计语言中定义的字体风格？
- [ ] **动效**：hover/active 效果是否符合设计语言？
- [ ] **阴影**：是否使用了设计语言中定义的阴影风格？
- [ ] **边框**：是否使用了设计语言中定义的边框风格？
- [ ] **差异化**：与默认主题相比，是否有明显的视觉差异？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
