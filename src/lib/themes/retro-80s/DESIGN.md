## 1. 设计故事 (Design Story)

**灵感来源**：
Retro 80s 主题的视觉灵感来自 1980 年代的电子文化：街机游戏（如 Pac-Man、Space Invaders）、MTV 音乐视频、霓虹灯招牌、合成器音乐封面、早期计算机界面（如 Commodore 64、Apple II）以及科幻电影（如《银翼杀手》、《电子世界争霸战》）。这是一个充满电子脉冲、像素艺术和霓虹色彩的年代，代表了数字文化的黎明。

**情感表达**：
用户应感受到**怀旧**、**活力**、**未来感**与**电子脉冲**。界面传达出一种"回到未来"的兴奋感，每个元素都像 80 年代的霓虹灯一样闪烁发光。这是一种"数字朋克"美学，强调高对比度、高饱和度和强烈的视觉冲击，让人想起那个充满可能性的年代。

**目标用户场景**：
适合电子游戏、音乐应用、创意作品集、独立开发者网站、复古风格应用、实验性 Web 项目、科技产品展示或任何需要传达"80年代未来感"和"数字怀旧"的场景。不适合需要专业、稳重感的商务应用。

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | **小圆角 (2-4px)** | 80年代界面介于硬边和圆润之间，小圆角保持现代感同时带有复古味道。某些元素（如像素风格按钮）可以使用直角。 |
| **是否使用切角/异形** | **是（可选）** | 可以使用像素风格的切角效果（clip-path），模拟 8-bit 游戏中的像素块，增加复古感。但不要过度使用，保持可读性。 |

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | **霓虹发光 + 颜色增强** | 元素 hover 时产生霓虹发光效果（box-shadow 发光），颜色饱和度增加，模拟 80 年代霓虹灯被激活的感觉。 |
| **Active 效果** | **快速闪烁 + 轻微缩放** | 按下时快速闪烁（颜色变化），轻微缩放（scale 0.98），模拟街机按钮被按下的反馈。 |
| **过渡时长** | **快速 (100-150ms)** | 快速、干脆的过渡，符合 80 年代电子设备的响应速度，拒绝拖泥带水。某些特殊效果（如扫描线）可以使用更快的动画。 |

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | **等宽字体 (Monospace)** | 使用系统等宽字体（如 'Courier New', 'Monaco', 'Consolas'），模拟 80 年代计算机终端和游戏界面的字体风格。这是 Retro 80s 的核心特征。 |
| **大小写** | **全大写 (Uppercase)** | 按钮、标签、重要文本使用全大写，增加视觉重量和电子感，符合 80 年代界面设计习惯。正文可以使用正常大小写。 |
| **字距** | **紧密 (Tight)** | 配合等宽字体和全大写使用紧密字距，营造紧凑、电子化的感觉，模拟早期计算机显示器的字符密度。 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | **霓虹发光阴影（彩色发光）** | 使用彩色发光阴影（如 `0 0 20px var(--primary)`），模拟霓虹灯效果。这是 80 年代美学的核心视觉特征，每个元素都像霓虹灯一样发光。 |
| **边框风格** | **粗实线（2-3px solid）+ 发光边框** | 使用粗实线边框（2-3px），颜色与元素主色相同，并配合发光效果。某些元素可以使用双边框（内边框 + 外发光）增加层次感。 |

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| Button | 2-4px | 2px solid + 发光 | 霓虹发光阴影 | 发光增强 + 颜色加深 | 闪烁 + scale(0.98) |
| Input | 2px | 2px solid + focus 发光 | 无或轻微发光 | border 发光 | focus 时强烈发光 |
| Card | 4px | 2px solid + 发光 | 霓虹发光阴影 | 发光增强 | - |
| Badge | 2px | 1px solid | 轻微发光 | 发光增强 | - |
| Dialog | 4px | 3px solid + 强烈发光 | 强烈霓虹发光 | - | - |
| Dropdown | 2px | 2px solid + 发光 | 霓虹发光阴影 | 发光增强 | - |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=primary, secondary=secondary, ghost=transparent │
│   │ Input        │ bg-surface-1, border-border, focus 时才用 primary       │
│   │ Card         │ bg-surface-1（容器用中性色，不要用 primary）             │
│   │ Dialog       │ bg-surface-1 或 bg-background                           │
│   │ Dropdown     │ bg-surface-2（悬浮层）                                   │
│   │ Table        │ bg-transparent 或 bg-muted（斑马纹）                     │
│   │ Tabs         │ 激活项才用 primary，未激活用 muted                        │
│   │ Badge        │ 根据语义：default=primary, destructive=红色              │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - Primary 只用于主要 CTA 和激活状态                                      │
│   │ - 容器类组件用 surface/background                                       │
│   │ - 表单组件用 surface + border，focus 时才用 primary                     │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | **#FF00FF (霓虹粉/洋红)** | 主要操作、强调、CTA 按钮 | 80年代最具代表性的霓虹色，充满活力和未来感 |
| `--secondary` | **#00FFFF (霓虹青/青色)** | 次要操作、信息展示 | 与粉红形成经典对比，模拟 CRT 显示器色彩 |
| `--accent` | **#FFFF00 (霓虹黄)** | 点缀、警告、吸引注意 | 高饱和度的黄色，增加视觉冲击 |

### 5.2 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | **#0A0A0A (深黑)** | 最底层，模拟 CRT 显示器的深色背景 |
| `--surface-1` | **#1A1A1A (深灰黑)** | 卡片、面板，略微亮于背景，保持层次 |
| `--surface-2` | **#252525 (中灰黑)** | 下拉菜单、悬浮层，更亮的层级 |
| `--surface-3` | **#333333 (浅灰黑)** | 最高层级，对话框等 |

### 5.3 色彩关系

**整体氛围**：深色背景 + 高饱和度霓虹色，形成强烈的对比和视觉冲击。

**色彩搭配原则**：
- **主色系**：霓虹粉（#FF00FF）作为主要 CTA，与深黑背景形成最强对比
- **辅助色系**：霓虹青（#00FFFF）用于次要操作和信息展示
- **强调色系**：霓虹黄（#FFFF00）用于警告和吸引注意
- **背景色系**：深黑到浅灰黑的渐变，保持层次感
- **文字色系**：白色/浅灰用于深色背景，黑色用于彩色背景（高对比度）

**特殊效果**：
- 所有彩色元素都配合发光阴影（glow effect）
- 可以使用渐变背景（如粉到紫的渐变）增加 80 年代感
- 某些元素可以使用扫描线效果（scanline）模拟 CRT 显示器

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了设计语言中定义的圆角/切角？
- [ ] **字体**：是否使用了设计语言中定义的字体风格？
- [ ] **动效**：hover/active 效果是否符合设计语言？
- [ ] **阴影**：是否使用了设计语言中定义的阴影风格？
- [ ] **边框**：是否使用了设计语言中定义的边框风格？
- [ ] **差异化**：与默认主题相比，是否有明显的视觉差异？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
