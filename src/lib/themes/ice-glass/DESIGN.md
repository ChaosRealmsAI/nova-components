## 1. 设计故事 (Design Story)

> 用 2-3 段话描述这个主题的灵感来源和情感表达

**灵感来源**：
此主题灵感来源于现代操作系统（如 macOS Big Sur, Windows 11 Fluent Design）中的毛玻璃（Frosted Glass）效果，以及科幻电影中的全息投影界面。通过模拟光线穿过冰层或磨砂玻璃的物理特性，创造出一种“凝固的空气”般的质感。

**情感表达**：
Ice Glass 旨在传递“轻盈”、“通透”与“未来感”。用户在使用时应感到界面元素悬浮于背景之上，操作反馈如同敲击冰晶般清脆（视觉上的清脆）。它消除厚重的色块，用光影和模糊来界定空间，带来冷静、理性的沉浸式体验。

**目标用户场景**：
非常适合数据可视化大屏、现代化的 IDE/编辑器、加密货币/Web3 应用 dashboard，以及任何追求高科技感和精致视觉体验的工具类软件。

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | 中等偏大圆角 (Radius-lg/xl) | 玻璃材质在自然界中很少呈现锐利的直角，圆角能增强“凝固液体”的有机感和柔和感，避免视觉割裂。 |
| **是否使用切角/异形** | 否 | 保持流体凝固的完整性，切角会破坏玻璃的表面张力感。 |

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | 亮度增加 (Brightness up) + 极微小的上浮 (TranslateY) | 模拟光线照在玻璃上的反光增强效果，以及物体悬浮的失重感。 |
| **Active 效果** | 亮度降低 + 缩放 (Scale down) | 模拟按压玻璃表面的物理反馈。 |
| **过渡时长** | 中速平滑 (300ms ease-out) | 玻璃是有“厚度”和“阻尼”的，动效不应过于跳跃，而应如液体般流淌。 |

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | Inter / SF Pro (System Sans) | 保持极简，不喧宾夺主，让内容像浮在玻璃上的印刷体。 |
| **大小写** | 标题大写 (Uppercase) + 宽字距 | 增加空气感和科技感，使标题更具结构性。 |
| **字距** | tracking-wide (for headers) | 增加呼吸感，配合通透的背景。 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | 柔和的彩色弥散光 (Glow) | 玻璃本身不产生黑色投影，而是折射光线。使用 colored shadow 模拟光线透过玻璃投射在背景上的辉光。 |
| **边框风格** | 1px 半透明高亮边框 (Glass Edge) | 模拟玻璃切面的高光（Rim Light），这是识别“玻璃感”的关键特征。 |

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| **Button** | Pill (全圆角) | 1px 亮白半透明 | 内部高光 (Inset) | 背景不透明度增加 + 辉光增强 | 缩小 0.95 |
| **Input** | lg (12px) | 底部高亮 (Underline style) 或 1px 全边框 | 无 | 边框变亮 | - |
| **Card** | xl (16px) | 1px 渐变边框 (上白下透) | 模糊投影 (Shadow-xl) | 微微上浮 | - |
| **Badge** | md (6px) | 无边框 | 内部微光 | - | - |
| **Dialog** | 2xl (24px) | 1px 亮白边框 | 强模糊背景 (Backdrop-blur-xl) | - | - |
| **Dropdown**| lg (12px) | 1px 极细边框 | 阴影 + 模糊 | 选项背景变亮 | - |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=Glass gradient, ghost=transparent               │
│   │ Input        │ bg-white/5 (ultra low opacity), border-white/10         │
│   │ Card         │ bg-white/5 or black/20 + backdrop-blur                  │
│   │ Dialog       │ bg-black/40 + backdrop-blur-md                          │
│   │ Dropdown     │ bg-black/60 + backdrop-blur-xl                          │
│   │ Table        │ header=transparent, row-hover=white/5                   │
│   │ Tabs         │ active=white/10 + glow, inactive=muted                  │
│   │ Badge        │ bg-primary/20 + text-primary-foreground                 │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - 尽量使用 Alpha 通道颜色 (rgba/hsla) 而不是实色                         │
│   │ - 利用 backdrop-filter: blur() 创造层级                                  │
│   │ - 边框通常比背景更亮（模拟反光）                                          │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `190 80% 60%` (Cyan/Ice Blue) | 激活状态、主要按钮、高光 | 科技、冷静、纯净 |
| `--secondary` | `210 50% 80%` (Soft Blue Grey) | 次要信息、辅助背景 | 柔和、支撑 |
| `--accent` | `300 70% 60%` (Neon Purple) | 强调、特殊提示、渐变对冲色 | 神秘、赛博 |

### 5.2 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `220 40% 10%` (Deep Space Blue) | 基础背景，建议带有噪点或极光图案 |
| `--surface-1` | `0 0% 100% / 0.05` (5% White) | 基础卡片，叠加 blur-md |
| `--surface-2` | `0 0% 100% / 0.10` (10% White) | 悬浮面板，叠加 blur-lg |
| `--surface-3` | `0 0% 100% / 0.20` (20% White) | 最高层级/高亮区域，叠加 blur-xl |

### 5.3 色彩关系

整体采用**深色模式**为主基调（因为发光和半透明效果在深色背景下最明显）。
色彩不是通过“填充”体现，而是通过“光晕”和“边缘”体现。
Primary 色主要用于文字高亮、边框辉光和按钮渐变。
背景色需要足够的深，以衬托前景玻璃的通透感。

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了设计语言中定义的圆角/切角？
- [ ] **字体**：是否使用了设计语言中定义的字体风格？
- [ ] **动效**：hover/active 效果是否符合设计语言？
- [ ] **阴影**：是否使用了设计语言中定义的阴影风格？
- [ ] **边框**：是否使用了设计语言中定义的边框风格？
- [ ] **差异化**：与默认主题相比，是否有明显的视觉差异？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
