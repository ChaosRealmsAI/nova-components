## 1. 设计故事 (Design Story)

**灵感来源**：
此主题灵感源自《银翼杀手》、《赛博朋克2077》等经典作品。它描绘了一个高科技与低生活共存的反乌托邦未来。霓虹灯光穿透黑暗的雨夜，故障艺术（Glitch Art）与工业机械感交织。

**情感表达**：
用户应感受到**科技感**、**锐利**、**叛逆**与**能量**。界面应当是高对比度的，带有强烈的视觉冲击力，传达出一种"未完成"或"被黑客入侵"的粗砺美感。

**目标用户场景**：
适合开发者工具、游戏界面、黑客马拉松展示页、Web3/区块链应用或任何需要强调前卫、极客属性的场景。

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | **直角 (0px) 或 切角 (Chamfered)** | 强调工业感和机械感，拒绝圆润的亲和力。主要容器使用切角设计。 |
| **是否使用切角/异形** | **是** | 使用 `clip-path` 实现 45 度切角，模拟芯片或机械面板的边缘。 |

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | **高亮发光 + 位移/故障** | 模拟电路通电瞬间的过载感。 |
| **Active 效果** | **瞬间下沉 + 颜色反转** | 机械按键的物理反馈，干脆利落。 |
| **过渡时长** | **极短 (75ms - 150ms)** | 强调电子信号的瞬时传输，拒绝拖泥带水。 |

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | **Sans-serif + Monospace** | 标题使用无衬线体保持现代感，正文和数据使用等宽字体模拟终端代码。 |
| **大小写** | **大写 (Uppercase)** | 按钮和标签使用全大写，增加警告感和严肃感。 |
| **字距** | **宽松 (Tracking-wide)** | 配合大写字母，增加呼吸感和科技排版风格。 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | **彩色霓虹辉光 (Neon Glow)** | 代替传统的黑色投影，使用主色调的模糊扩散模拟发光管。 |
| **边框风格** | **细实线 (1px) + 亮色描边** | 定义清晰的边界，Hover 时边框变亮或变色。 |

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| Button | 切角 (Clip) | 1px Solid Primary | Neon Glow | 背景填充 / 边框高亮 | 缩放 / 颜色反转 |
| Input | 直角 (0px) | 1px Solid Muted | None | 边框变 Primary | 边框变 Secondary |
| Card | 切角 (Clip) | 1px Solid Border | Box Shadow Glow | 边框发光 | N/A |
| Badge | 直角 (0px) | 1px Solid | Glow (sm) | Opacity 变化 | N/A |
| Dialog | 切角 (Clip) | 2px Solid Accent | Strong Glow | N/A | N/A |
| Dropdown | 直角 (0px) | 1px Solid Border | None | Item 背景变 Primary | Item 反转 |

## 5. 色彩设计 (Color Design)

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `#00e5ff` (Neon Cyan) | 主要操作、强调 | 冷静、科技、未来 |
| `--secondary` | `#d600ff` (Neon Purple) | 次要操作 | 神秘、能量 |
| `--accent` | `#fcee0a` (Neon Yellow) | 点缀、吸引注意 | 警告、危险、高亮 |

### 5.2 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `#050505` | 最底层，接近纯黑 |
| `--surface-1` | `#0f0f13` | 卡片、面板，深蓝黑 |
| `--surface-2` | `#1a1a2e` | 下拉菜单、悬浮，稍微提亮 |
| `--surface-3` | `#2a2a40` | 最高层级，强调区 |

### 5.3 色彩关系

整体采用**暗黑模式**基调。背景极深，以突出前景的霓虹色。
文字主要使用灰白色 (`#e0e0e0`) 保证可读性，次要文字使用冷灰色 (`#94a3b8`)。
边框色默认较暗 (`#334155`)，Hover 或 Active 时瞬间点亮为 Neon 色。
状态色（红绿黄）也均采用高饱和度的霓虹版本。

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了设计语言中定义的圆角/切角？
- [ ] **字体**：是否使用了设计语言中定义的字体风格？
- [ ] **动效**：hover/active 效果是否符合设计语言？
- [ ] **阴影**：是否使用了设计语言中定义的阴影风格？
- [ ] **边框**：是否使用了设计语言中定义的边框风格？
- [ ] **差异化**：与默认主题相比，是否有明显的视觉差异？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
