## 1. 设计故事 (Design Story)

**灵感来源**：
Neo Brutalism（新粗野主义）设计风格源于 20 世纪 50-70 年代的建筑运动，强调"形式追随功能"的原始美学。在数字设计中，它表现为高对比度、粗边框、硬边阴影和鲜艳色彩的组合。灵感来自瑞士国际主义平面设计、Bauhaus 风格以及现代网页设计中的"Brutalist Web Design"运动。

**情感表达**：
用户应感受到**直接**、**大胆**、**自信**与**原始力量**。界面拒绝圆滑和装饰，用最直接的方式传达信息。每个元素都像建筑中的混凝土块，厚重、清晰、不容忽视。这是一种"反精致"的美学，强调功能性和可读性，传达出一种"不妥协"的态度。

**目标用户场景**：
适合创意作品集、独立设计师网站、艺术展览页面、实验性 Web 应用、设计工具界面或任何需要传达"原始力量"和"直接沟通"的场景。不适合需要亲和力、温暖感的商业应用。

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | **直角 (0px)** | 粗野主义拒绝圆润，所有元素都是硬边矩形，强调几何感和结构感。 |
| **是否使用切角/异形** | **否** | 保持纯粹的矩形，不使用切角或异形，强调"原始"和"直接"。 |

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | **颜色加深 + 阴影保持** | 背景色略微加深，但保持硬边阴影不变，模拟"准备按下"的状态。 |
| **Active 效果** | **按下效果（translate + shadow 消失）** | 元素向下向右移动（2px 2px），阴影消失，模拟物理按键被按下的感觉。这是粗野主义的标志性交互。 |
| **过渡时长** | **极短 (75ms)** | 快速、干脆的反馈，拒绝拖泥带水，强调直接和果断。 |

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | **Sans-serif（系统字体）** | 使用系统默认无衬线字体，强调功能性和可读性，拒绝装饰性字体。 |
| **大小写** | **大写 (Uppercase)** | 按钮、标签、标题使用全大写，增加视觉重量和严肃感，符合粗野主义的"直接"美学。 |
| **字距** | **宽松 (Tracking-wide)** | 配合大写字母使用宽松字距，增加呼吸感和现代排版风格。 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | **硬边偏移阴影（无模糊）** | 使用纯黑色、无模糊的偏移阴影（如 4px 4px 0px 0px），模拟元素"浮起"的效果。这是粗野主义的核心视觉特征。 |
| **边框风格** | **粗实线（2px solid black）** | 所有元素都有 2px 的黑色实线边框，定义清晰的边界，增加视觉重量。 |

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| Button | 0px | 2px solid black | 4px 4px 0px 0px | 颜色加深 | translate(2px,2px) + shadow 消失 |
| Input | 0px | 2px solid black | 4px 4px 0px 0px | border 保持 | focus: ring-2 black |
| Card | 0px | 2px solid black | 4px 4px 0px 0px | N/A | N/A |
| Badge | 0px | 2px solid black | 2px 2px 0px 0px | 颜色加深 | N/A |
| Dialog | 0px | 2px solid black | 8px 8px 0px 0px | N/A | N/A |
| Dropdown | 0px | 2px solid black | 4px 4px 0px 0px | item: bg-accent | item: translate(1px,1px) |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=primary, secondary=secondary, ghost=transparent │
│   │ Input        │ bg-surface-1, border-border, focus 时才用 primary       │
│   │ Card         │ bg-surface-1（容器用中性色，不要用 primary）             │
│   │ Dialog       │ bg-surface-1 或 bg-background                           │
│   │ Dropdown     │ bg-surface-2（悬浮层）                                   │
│   │ Table        │ bg-transparent 或 bg-muted（斑马纹）                     │
│   │ Tabs         │ 激活项才用 primary，未激活用 muted                        │
│   │ Badge        │ 根据语义：default=primary, destructive=红色              │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - Primary 只用于主要 CTA 和激活状态                                      │
│   │ - 容器类组件用 surface/background                                       │
│   │ - 表单组件用 surface + border，focus 时才用 primary                     │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `#FF6B6B` (红色) | 主要操作、强调、破坏性操作 | 强烈、警告、行动 |
| `--secondary` | `#4ECDC4` (青色) | 次要操作、辅助按钮 | 冷静、平衡 |
| `--accent` | `#FFE66D` (黄色) | 点缀、吸引注意、hover 状态 | 活力、能量 |

### 5.2 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `#FFFFFF` (纯白) | 最底层，页面背景 |
| `--surface-1` | `#FFFFFF` (纯白) | 卡片、面板、输入框背景 |
| `--surface-2` | `#FFFFFF` (纯白) | 下拉菜单、悬浮层 |
| `--surface-3` | `#FFFFFF` (纯白) | 最高层级（对话框等） |

### 5.3 色彩关系

**高对比度原则**：
- 所有文字使用纯黑色 (`#000000`)，确保在白色背景上达到最高对比度
- 所有边框使用纯黑色 (`#000000`)，2px 粗细
- 背景使用纯白色 (`#FFFFFF`)，拒绝灰色渐变
- 功能色（primary/secondary/accent）使用高饱和度颜色，与黑色形成强烈对比

**整体氛围**：
- 强烈的视觉冲击力，拒绝"柔和"和"渐变"
- 每个元素都像建筑中的混凝土块，厚重、清晰、不容忽视
- 色彩用于功能区分，而非装饰

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了设计语言中定义的圆角/切角？
- [ ] **字体**：是否使用了设计语言中定义的字体风格？
- [ ] **动效**：hover/active 效果是否符合设计语言？
- [ ] **阴影**：是否使用了设计语言中定义的阴影风格？
- [ ] **边框**：是否使用了设计语言中定义的边框风格？
- [ ] **差异化**：与默认主题相比，是否有明显的视觉差异？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
