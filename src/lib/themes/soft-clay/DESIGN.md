## 1. 设计故事 (Design Story)

> 用 2-3 段话描述这个主题的灵感来源和情感表达

**灵感来源**：
新拟物主义 (Neumorphism) 设计趋势，融合了陶土、软泥的触感质感。
灵感来自手工陶艺的温润触感、北欧极简家居的柔和线条、以及 iOS 系统的精致阴影。
想象在温暖的午后阳光下，柔和的光线从窗口洒入，在物体表面形成细腻的明暗过渡。

**情感表达**：
用户应该感受到：**温暖、舒适、可触摸、宁静、专注**。
界面元素仿佛从背景中轻轻浮起，又仿佛可以被轻轻按下。
这是一种安静、不张扬的美，让用户专注于内容而非界面本身。

**目标用户场景**：
- 效率工具（笔记、待办、日程）
- 健康/冥想类应用
- 个人知识管理
- 阅读类应用
- 极简主义设计偏好的用户

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | 大圆角 (16-24px) | 柔和、有机的形态，符合陶土的圆润触感 |
| **是否使用切角/异形** | ❌ 否 | 保持有机的圆润感，避免锐利几何形状 |

**圆角规范**：
```
小组件（Badge, Switch）: 全圆角 (9999px / pill)
中组件（Button, Input）: 16px
大组件（Card, Dialog）: 24px
```

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | 阴影增强（浮起感） | 模拟物体被抬起，光影加深 |
| **Active 效果** | 内凹阴影（按压感） | 模拟物体被按入表面 |
| **过渡时长** | 200ms (ease-in-out) | 略慢于标准，创造柔和、有机的感觉 |

**核心动效 CSS**：
```css
/* 默认凸起状态 */
.neumorphic-raised {
  box-shadow:
    6px 6px 12px var(--shadow-dark),
    -6px -6px 12px var(--shadow-light);
}

/* Hover 增强凸起 */
.neumorphic-raised:hover {
  box-shadow:
    8px 8px 16px var(--shadow-dark),
    -8px -8px 16px var(--shadow-light);
  transform: translateY(-2px);
}

/* Active 内凹 */
.neumorphic-raised:active {
  box-shadow:
    inset 4px 4px 8px var(--shadow-dark),
    inset -4px -4px 8px var(--shadow-light);
  transform: translateY(0);
}
```

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | 系统无衬线 (rounded 优先) | 圆润的字体与圆润的形状呼应 |
| **大小写** | 正常大小写 | 保持阅读舒适度 |
| **字距** | 正常 | 不刻意强调，让内容自然流动 |
| **字重** | 标题 medium/semibold，正文 regular | 柔和的对比，不过于强烈 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | 双向柔和阴影（凸起/内凹） | 新拟物主义核心特征 |
| **边框风格** | ❌ 无边框 | 完全依赖阴影表达层次，保持纯净 |

**层次系统**：
```
┌─────────────────────────────────────────────────────────────────┐
│  新拟物主义层次原理：                                             │
│                                                                 │
│     光源方向：左上 ↘                                             │
│                                                                 │
│     凸起 (Raised)：                                              │
│     ┌─────────────┐                                             │
│     │ 亮 (左上)    │ ← 光照面                                    │
│     │             │                                             │
│     │    暗 (右下) │ ← 阴影面                                    │
│     └─────────────┘                                             │
│                                                                 │
│     内凹 (Inset)：                                               │
│     ┌─────────────┐                                             │
│     │ 暗 (左上)    │ ← 内部阴影                                  │
│     │             │                                             │
│     │    亮 (右下) │ ← 内部反光                                  │
│     └─────────────┘                                             │
└─────────────────────────────────────────────────────────────────┘
```

**阴影 Token 定义**：
```css
--shadow-dark: rgba(163, 156, 148, 0.5);   /* 右下阴影 */
--shadow-light: rgba(255, 255, 255, 0.8);  /* 左上高光 */
```

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角 | 边框 | 阴影 | Hover | Active |
|----------|------|------|------|-------|--------|
| **Button** | 16px | 无 | 凸起双阴影 | 阴影加深+上移 | 内凹阴影 |
| **Input** | 16px | 无 | 内凹阴影（始终凹入） | 内阴影加深 | - |
| **Card** | 24px | 无 | 凸起双阴影 | 阴影加深 | - |
| **Badge** | 全圆角 (pill) | 无 | 轻微凸起 | 阴影加深 | - |
| **Dialog** | 24px | 无 | 强凸起阴影 | - | - |
| **Dropdown** | 16px | 无 | 凸起阴影 | 选项背景渐变 | 内凹 |
| **Switch** | 全圆角 (pill) | 无 | 轨道内凹，滑块凸起 | - | 滑块内凹 |
| **Checkbox** | 8px | 无 | 内凹阴影 | 内阴影加深 | 凸起（选中） |
| **Tabs** | 16px | 无 | 激活项凸起 | 轻微凸起 | 内凹 |
| **Slider** | 全圆角 | 无 | 轨道内凹，滑块凸起 | 滑块阴影加深 | 滑块内凹 |
| **Progress** | 全圆角 | 无 | 轨道内凹，进度条凸起 | - | - |
| **Avatar** | 全圆角 | 无 | 轻微凸起 | 阴影加深 | - |
| **Tooltip** | 12px | 无 | 柔和凸起 | - | - |

**特殊组件说明**：

| 组件 | 特殊效果 |
|------|----------|
| **Input** | 始终使用内凹阴影，模拟凹槽输入区 |
| **Switch** | 轨道内凹 + 滑块凸起，形成开关的触感 |
| **Skeleton** | 柔和的脉冲动画，模拟呼吸感 |
| **Separator** | 使用内凹线条效果，而非实线 |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=primary, secondary=secondary, ghost=transparent │
│   │ Input        │ bg-surface-1, border-border, focus 时才用 primary       │
│   │ Card         │ bg-surface-1（容器用中性色，不要用 primary）             │
│   │ Dialog       │ bg-surface-1 或 bg-background                           │
│   │ Dropdown     │ bg-surface-2（悬浮层）                                   │
│   │ Table        │ bg-transparent 或 bg-muted（斑马纹）                     │
│   │ Tabs         │ 激活项才用 primary，未激活用 muted                        │
│   │ Badge        │ 根据语义：default=primary, destructive=红色              │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - Primary 只用于主要 CTA 和激活状态                                      │
│   │ - 容器类组件用 surface/background                                       │
│   │ - 表单组件用 surface + border，focus 时才用 primary                     │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `#6B8E9F` (雾蓝) | 主要 CTA、激活状态 | 宁静、专注、信赖 |
| `--secondary` | `#9F8B7B` (陶土) | 次要操作 | 温暖、自然、接地 |
| `--accent` | `#B5A08E` (沙棕) | 点缀 | 柔和、舒适 |

### 5.2 语义色

| Token | 色值 | 用途 |
|-------|------|------|
| `--success` | `#7BA68D` | 成功、完成 - 柔和的薄荷绿 |
| `--warning` | `#C9A66B` | 警告、注意 - 柔和的琥珀 |
| `--error` | `#B87878` | 错误、危险 - 柔和的玫瑰红 |
| `--destructive` | `#B87878` | 删除操作 - 与 error 相同 |
| `--info` | `#7B9EB5` | 信息提示 - 柔和的天蓝 |

### 5.3 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `#E8E4DF` | 最底层 - 暖灰陶土色 |
| `--surface-1` | `#E8E4DF` | 与背景同色（靠阴影区分） |
| `--surface-2` | `#ECE8E3` | 下拉菜单 - 略浅 |
| `--surface-3` | `#F0ECE7` | 最高层级 - 更浅 |

**⚠️ 新拟物主义特殊说明**：
- `surface-1` 与 `background` **同色**，完全依靠阴影区分层级
- 这是新拟物主义的核心特征：元素与背景融为一体

### 5.4 文字色

| Token | 色值 | 用途 |
|-------|------|------|
| `--foreground` | `#4A4543` | 主要文字 - 深暖灰 |
| `--foreground-secondary` | `#6B6562` | 次要文字 |
| `--muted-foreground` | `#8A847F` | 柔和文字、描述 |
| `--subtle-foreground` | `#A5A09B` | 占位符、禁用 |

### 5.5 阴影色（新拟物主义核心）

| Token | 色值 | 用途 |
|-------|------|------|
| `--shadow-dark` | `rgba(163, 156, 148, 0.5)` | 右下阴影 |
| `--shadow-light` | `rgba(255, 255, 255, 0.8)` | 左上高光 |
| `--shadow-inset-dark` | `rgba(163, 156, 148, 0.4)` | 内凹阴影 |
| `--shadow-inset-light` | `rgba(255, 255, 255, 0.7)` | 内凹高光 |

### 5.6 色彩关系

```
                    ┌─────────────────────────────────────────────────────┐
                    │                 Soft Clay 色彩系统                    │
                    └─────────────────────────────────────────────────────┘

    背景层（统一色调）                 功能色（柔和变体）              阴影系统
    ─────────────────               ───────────────────            ──────────

    ┌──────────────┐                ┌──────────────┐              ┌──────────────┐
    │   #E8E4DF    │ background     │   #6B8E9F    │ primary      │ 右下阴影      │
    │   暖灰陶土    │ = surface-1    │   雾蓝        │ CTA/激活     │ rgba(163...)  │
    ├──────────────┤                ├──────────────┤              ├──────────────┤
    │   #ECE8E3    │ surface-2      │   #9F8B7B    │ secondary    │ 左上高光      │
    │   略浅        │ 悬浮层         │   陶土        │ 次要         │ rgba(255...)  │
    ├──────────────┤                ├──────────────┤              └──────────────┘
    │   #F0ECE7    │ surface-3      │   #7BA68D    │ success
    │   更浅        │ 最高层         │   薄荷绿      │ 成功
    └──────────────┘                └──────────────┘

    设计原则：
    • 所有颜色都是柔和、低饱和度的
    • 背景与表面同色，依靠阴影区分
    • 整体氛围：温暖、宁静、触感
    • 避免高对比度，追求和谐统一
```

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了大圆角 (16-24px) 或全圆角？
- [ ] **字体**：是否使用了柔和的字重（medium 而非 bold）？
- [ ] **动效**：hover 是否有凸起增强？active 是否有内凹效果？
- [ ] **阴影**：是否使用了双向阴影（暗+亮）？是否避免了边框？
- [ ] **边框**：是否完全依赖阴影而非边框？
- [ ] **差异化**：与默认主题对比，是否有明显的新拟物主义风格？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 7. 可复用样式代码片段

开发组件时，可直接使用以下代码片段：

### 7.1 凸起效果 (Raised)

```css
/* 默认凸起 - 用于 Button, Card */
.neu-raised {
  box-shadow:
    6px 6px 12px var(--shadow-dark),
    -6px -6px 12px var(--shadow-light);
}

/* 小凸起 - 用于 Badge, 小按钮 */
.neu-raised-sm {
  box-shadow:
    3px 3px 6px var(--shadow-dark),
    -3px -3px 6px var(--shadow-light);
}

/* 强凸起 - 用于 Dialog */
.neu-raised-lg {
  box-shadow:
    10px 10px 20px var(--shadow-dark),
    -10px -10px 20px var(--shadow-light);
}
```

### 7.2 内凹效果 (Inset)

```css
/* 内凹 - 用于 Input, Checkbox, Switch 轨道 */
.neu-inset {
  box-shadow:
    inset 4px 4px 8px var(--shadow-dark),
    inset -4px -4px 8px var(--shadow-light);
}

/* 小内凹 */
.neu-inset-sm {
  box-shadow:
    inset 2px 2px 4px var(--shadow-dark),
    inset -2px -2px 4px var(--shadow-light);
}
```

### 7.3 Hover 增强

```css
/* Hover 凸起增强 */
.neu-hover-raised:hover {
  box-shadow:
    8px 8px 16px var(--shadow-dark),
    -8px -8px 16px var(--shadow-light);
  transform: translateY(-2px);
}
```

### 7.4 Active 按压

```css
/* Active 内凹（按压） */
.neu-active-inset:active {
  box-shadow:
    inset 4px 4px 8px var(--shadow-dark),
    inset -4px -4px 8px var(--shadow-light);
  transform: translateY(0);
}
```

### 7.5 圆角规范

```css
/* 小组件 - Badge, Switch, Progress */
.radius-pill { border-radius: 9999px; }

/* 中组件 - Button, Input, Dropdown */
.radius-md { border-radius: 16px; }

/* 大组件 - Card, Dialog */
.radius-lg { border-radius: 24px; }

/* 小圆角 - Checkbox */
.radius-sm { border-radius: 8px; }
```

### 7.6 过渡动画

```css
/* 标准过渡 - 略慢于默认，更柔和 */
.transition-neu {
  transition: all 200ms ease-in-out;
}
```

### 7.7 分隔线（内凹线）

```css
/* 新拟物主义分隔线 */
.neu-separator {
  height: 2px;
  background: linear-gradient(
    to bottom,
    var(--shadow-dark),
    var(--shadow-light)
  );
  border: none;
}
```
