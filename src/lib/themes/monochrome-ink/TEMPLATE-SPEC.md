# 主题模板编写规范

本目录是主题开发的模板源，用于 `pnpm create:theme` 命令复制。

---

## 核心理念

```
模板 = 结构(固定) + 用途(说明) + 约束(必须满足) + 自由发挥(鼓励创意)
```

**模板不是示例代码**，只告诉开发者：
1. 这个位置应该放什么
2. 必须满足什么条件
3. 可以怎样自由发挥

---

## 组件模板格式

### 文件头部

```typescript
/**
 * {ComponentName} 组件样式模板
 *
 * ═══════════════════════════════════════════════════════════════════════════
 * 重要原则：
 * 1. 根据主题风格灵活发挥，没有固定写法
 * 2. Token 使用规范请参考 SOP 文档
 * 3. 只要满足"用途约束"，样式实现完全自由
 * ═══════════════════════════════════════════════════════════════════════════
 */
```

### Slot 格式

```typescript
/**
 * {slotName}: {一句话描述}
 * ─────────────────────────────────────────────────────────────────────
 * {详细说明}
 *
 * 自由发挥：
 * - {可能的样式类型}
 * - ...
 */
{slotName}: [
  // TODO: 根据主题风格自由发挥
],
```

### Variant 格式

```typescript
/**
 * {variantValue}: {一句话描述}
 * ───────────────────────────────────────────────────────────────────
 * 用途：{什么场景使用}
 * 约束：{必须满足的条件}
 *
 * 自由发挥：{可选的效果}
 */
{variantValue}: {
  base: [
    // TODO
  ],
},
```

## Tokens 模板格式

```typescript
/**
 * {分类名} - {数量}个
 */

/**
 * {tokenName}: {用途说明}
 */
'--{token-name}': '',  // TODO
```

## Playground 模板格式

```typescript
/**
 * {分类名}
 * ════════════════════════════════════════════════════════════════════
 */

/**
 * {变量说明}
 * 自由发挥：{可能的值}
 */
--pg-{name}: ;  /* TODO */
```

## 分隔线规范

```typescript
// 文件级别（最粗）
// ═══════════════════════════════════════════════════════════════════════════

// Section 级别
// ============================================================================

// Slot/Variant 级别
// ─────────────────────────────────────────────────────────────────────
```

---

## 要点

1. **不写具体样式值** - 只写 `// TODO` 或空字符串
2. **强调"自由发挥"** - 鼓励开发者根据主题风格创作
3. **明确"约束"** - 如必须使用哪个 Token
4. **说明"用途"** - 这个 slot/variant 是干什么的
