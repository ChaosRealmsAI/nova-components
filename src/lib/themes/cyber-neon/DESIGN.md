## 1. 设计故事 (Design Story)

> 用 2-3 段话描述这个主题的灵感来源和情感表达

**灵感来源**：
东京涩谷的霓虹街道、《银翼杀手》《攻壳机动队》的赛博朋克美学、80年代电子游戏厅的霓虹灯光。
深邃的黑色背景如同午夜的都市天际线，而霓虹色彩如同穿透黑暗的光剑——粉色代表活力与叛逆，
青色代表科技与未来，两者交织形成强烈的视觉冲击。

**情感表达**：
用户应该感受到：**未来感、科技感、酷炫、叛逆、充满能量**。
就像置身于一个高科技低生活的赛博朋克世界，每一个交互都带有电子的脉动感。
这不是冷冰冰的科技，而是充满生命力的霓虹之光。

**目标用户场景**：
- 游戏类应用（游戏启动器、游戏社区、电竞平台）
- 音乐/DJ 类应用
- 科技产品展示网站
- 开发者工具（终端、IDE 主题）
- 加密货币/Web3 应用
- 创意作品集

## 2. 设计语言 (Design Language)

### 2.1 形状语言 (Shape Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **圆角风格** | 小圆角 (4px) 或切角 | 锐利的边缘传达科技感和未来感，但不完全尖锐以保持可用性 |
| **是否使用切角/异形** | ✅ 是 - 对角切角 (chamfer) | 经典赛博朋克元素，如同高科技硬件的倒角设计 |

**切角 CSS 实现**：
```css
/* 按钮/卡片的切角效果 */
clip-path: polygon(
  8px 0, 100% 0, 100% calc(100% - 8px),
  calc(100% - 8px) 100%, 0 100%, 0 8px
);
```

### 2.2 动效语言 (Motion Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **Hover 效果** | 霓虹发光增强 + 轻微向上位移 (translateY: -1px) | 模拟霓虹灯被激活时的光晕扩散效果 |
| **Active 效果** | 光晕收缩 + 轻微下压 + 颜色加亮 | 模拟按下按钮时电流通过的反馈 |
| **过渡时长** | 150ms (ease-out) | 快速响应体现科技的高效 |
| **特殊效果** | 可选：扫描线动画、闪烁效果 | 增强赛博朋克氛围（慎用，避免干扰） |

**发光效果 CSS**：
```css
/* 霓虹发光 */
box-shadow:
  0 0 5px var(--primary),
  0 0 10px var(--primary),
  0 0 20px color-mix(in srgb, var(--primary) 50%, transparent);

/* Hover 增强 */
box-shadow:
  0 0 8px var(--primary),
  0 0 16px var(--primary),
  0 0 32px color-mix(in srgb, var(--primary) 60%, transparent);
```

### 2.3 字体语言 (Typography Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **主字体** | 系统无衬线 / 可选等宽字体用于标签 | 无衬线保持现代感，等宽字体增加科技感 |
| **大小写** | 按钮/徽章可用全大写 (uppercase)，正文正常 | 大写字母更有力量感和未来感 |
| **字距** | 按钮/标签加宽 (0.05em)，正文正常 | 加宽字距增强科技感和可读性 |
| **字重** | 标题用 semibold/bold，正文用 regular | 强对比体现层级 |

### 2.4 层次语言 (Hierarchy Language)

| 维度 | 你的选择 | 设计理由 |
|------|----------|----------|
| **阴影风格** | 霓虹发光 (glow) 而非传统阴影 | 发光是霓虹的本质特征 |
| **边框风格** | 1px 发光边框，激活时加粗 | 边框如同霓虹管，承载光芒 |
| **层级表达** | 发光强度表示层级（越高越亮） | 高层级 = 更亮的发光 |

**边框发光 CSS**：
```css
/* 静态边框 */
border: 1px solid var(--primary);
box-shadow:
  inset 0 0 10px color-mix(in srgb, var(--primary) 10%, transparent),
  0 0 5px color-mix(in srgb, var(--primary) 30%, transparent);

/* Focus 状态 */
border: 1px solid var(--primary);
box-shadow:
  0 0 0 1px var(--primary),
  0 0 10px var(--primary);
```

## 3. 底线原则（必须遵守）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔴 可读性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 文字与背景对比度必须 ≥ 4.5:1 (WCAG AA)                                    │
│  • 主要内容文字不小于 14px                                                    │
│  • 禁止使用纯装饰性效果遮挡内容                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 比例协调原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 内边距使用 4px 倍数 (p-1=4px, p-2=8px, p-4=16px)                          │
│  • 圆角不超过元素高度的 50%                                                   │
│  • 图标与文字大小协调 (图标 = 文字大小 × 1.2~1.5)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 一致性原则                                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • 同类组件使用相同的设计语言                                                 │
│  • 同一 variant 在不同组件中语义一致                                          │
│  • 动效时长在整个主题中保持一致                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 状态清晰可辨原则                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • hover 状态必须有明显视觉变化                                               │
│  • active 状态必须有按压反馈                                                  │
│  • focus 状态必须有清晰的焦点环                                               │
│  • disabled 状态必须明显区别于正常状态                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔴 语义正确原则                                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  • destructive 必须使用 destructive Token（红色系）                          │
│  • success 必须传达积极含义（绿色系）                                         │
│  • warning 必须引起注意（黄色/橙色系）                                        │
│  • primary 是最强视觉权重                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Slot 级别差异化要求

**⭐ 这是最重要的部分！**

你拥有的能力（必须充分发挥）：

| 维度 | 你可以做什么 |
|------|-------------|
| **形状** | 圆角大小、切角、不规则形状、clip-path 任意裁剪 |
| **字体** | 字族、大小写、字重、字距、行高 |
| **动效** | hover 变化、active 反馈、过渡时长、动画曲线 |
| **阴影** | 无阴影、柔和、硬边、发光、内凹、多层 |
| **边框** | 粗细、样式、颜色、发光、渐变 |
| **背景** | 纯色、渐变、半透明、模糊、图案 |

**如果你的组件只是换了颜色，说明没有发挥 Slot 能力。**

### 组件风格一览表

填写此表以确保设计一致性：

| 组件类型 | 圆角/形状 | 边框 | 阴影/发光 | Hover | Active |
|----------|-----------|------|-----------|-------|--------|
| **Button** | 切角 (chamfer) | 1px 霓虹边框 | 霓虹发光 | 发光增强+上移 | 发光收缩+下压 |
| **Input** | 4px 小圆角 | 1px 边框 (muted) | 无 → focus 时发光 | 边框变亮 | 内发光 |
| **Card** | 切角 (chamfer) | 1px 霓虹边框 | 柔和发光 | 发光增强 | - |
| **Badge** | 4px 小圆角 | 1px 发光边框 | 轻微发光 | 发光增强 | - |
| **Dialog** | 切角 (大) | 1px 霓虹边框 | 强发光 | - | - |
| **Dropdown** | 4px 小圆角 | 1px 霓虹边框 | 柔和发光 | 选项背景高亮 | 发光边框 |
| **Table** | 无圆角 | 1px 底边框 | 无 | 行背景高亮 | - |
| **Tabs** | 无圆角 | 底部2px指示线 | 激活项发光 | 文字变亮 | 指示线发光 |
| **Switch** | 全圆角 (pill) | 1px 边框 | 激活时发光 | 发光增强 | - |
| **Checkbox** | 4px 小圆角 | 1px 边框 | 选中时发光 | 边框变亮 | 内发光 |
| **Tooltip** | 4px 小圆角 | 1px 边框 | 轻微发光 | - | - |
| **Progress** | 全圆角 (pill) | 无 | 进度条发光 | - | - |

**特殊组件说明**：

| 组件 | 特殊效果 |
|------|----------|
| **Alert** | 左侧 3px 发光边框作为类型指示器 |
| **Avatar** | 外圈发光环，在线状态用绿色发光点 |
| **Skeleton** | 扫描线动画（从左到右的光线扫过） |
| **Loading** | 霓虹色旋转光环 |

## 5. 色彩设计 (Color Design)

### 5.0 颜色使用原则（重要！）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   ⛔ 不要所有组件都用 primary！                                              │
│                                                                             │
│   颜色分配原则：                                                             │
│                                                                             │
│   │ 组件类型     │ 应该用什么颜色                                           │
│   │──────────────│─────────────────────────────────────────────────────────│
│   │ Button       │ default=primary, secondary=secondary, ghost=transparent │
│   │ Input        │ bg-surface-1, border-border, focus 时才用 primary       │
│   │ Card         │ bg-surface-1（容器用中性色，不要用 primary）             │
│   │ Dialog       │ bg-surface-1 或 bg-background                           │
│   │ Dropdown     │ bg-surface-2（悬浮层）                                   │
│   │ Table        │ bg-transparent 或 bg-muted（斑马纹）                     │
│   │ Tabs         │ 激活项才用 primary，未激活用 muted                        │
│   │ Badge        │ 根据语义：default=primary, destructive=红色              │
│   │                                                                         │
│   │ ⭐ 核心原则：                                                            │
│   │ - Primary 只用于主要 CTA 和激活状态                                      │
│   │ - 容器类组件用 surface/background                                       │
│   │ - 表单组件用 surface + border，focus 时才用 primary                     │
│   │                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.1 主色调

| Token | 色值 | 用途 | 情感 |
|-------|------|------|------|
| `--primary` | `#ff2d95` (霓虹粉) | 主要 CTA、激活状态、强调 | 活力、叛逆、热情 |
| `--secondary` | `#00f0ff` (霓虹青) | 次要操作、链接、信息 | 科技、冷静、未来 |
| `--accent` | `#b94fff` (霓虹紫) | 点缀、特殊状态 | 神秘、创意 |

### 5.2 语义色

| Token | 色值 | 用途 |
|-------|------|------|
| `--destructive` | `#ff3b3b` | 删除、错误、危险操作 |
| `--success` | `#00ff88` | 成功、完成、在线状态 |
| `--warning` | `#ffaa00` | 警告、注意 |
| `--muted` | `#6b7280` | 禁用、次要文字 |

### 5.3 背景层级

| Token | 色值 | 层级说明 |
|-------|------|----------|
| `--background` | `#0a0a0f` | 最底层 - 深邃的夜空 |
| `--surface-1` | `#12121a` | 卡片、面板 - 略微提升 |
| `--surface-2` | `#1a1a25` | 下拉菜单、悬浮层 |
| `--surface-3` | `#222230` | 最高层级 - 模态框 |

### 5.4 文字色

| Token | 色值 | 用途 |
|-------|------|------|
| `--foreground` | `#f0f0f5` | 主要文字 - 近乎纯白 |
| `--foreground-muted` | `#8888a0` | 次要文字、描述 |
| `--foreground-subtle` | `#555566` | 占位符、禁用文字 |

### 5.5 边框色

| Token | 色值 | 用途 |
|-------|------|------|
| `--border` | `#2a2a3a` | 默认边框 - 低调 |
| `--border-hover` | `#3a3a4a` | Hover 时边框 |
| `--border-focus` | `var(--primary)` | Focus 时使用主色 |

### 5.6 色彩关系

```
                    ┌─────────────────────────────────────────────────────┐
                    │                    色彩系统                          │
                    └─────────────────────────────────────────────────────┘

    深色背景层                      霓虹前景色                     语义色
    ──────────                    ────────────                  ────────

    ┌──────────┐                  ┌──────────┐                 ┌──────────┐
    │ #0a0a0f  │ background       │ #ff2d95  │ primary        │ #ff3b3b  │ destructive
    ├──────────┤                  │ 霓虹粉    │ 主CTA/激活      │ 霓虹红    │ 删除/错误
    │ #12121a  │ surface-1        ├──────────┤                 ├──────────┤
    ├──────────┤                  │ #00f0ff  │ secondary       │ #00ff88  │ success
    │ #1a1a25  │ surface-2        │ 霓虹青    │ 次要/信息       │ 霓虹绿    │ 成功/在线
    ├──────────┤                  ├──────────┤                 ├──────────┤
    │ #222230  │ surface-3        │ #b94fff  │ accent          │ #ffaa00  │ warning
    └──────────┘                  │ 霓虹紫    │ 点缀           │ 霓虹橙    │ 警告
                                  └──────────┘                 └──────────┘

    对比关系：
    • 背景极暗 (L: ~3-8%) ← → 前景极亮 (霓虹色 L: 50-70%)
    • 高对比度确保可读性，霓虹色在暗背景上自然发光

    色彩氛围：
    • 粉青双主色形成冷暖对比，增加视觉张力
    • 紫色作为过渡色，连接粉与青
    • 整体传达：夜晚、科技、能量、未来
```

## 6. 设计检查清单

在开发组件时，逐项检查：

### 6.1 每个组件必检项

- [ ] **形状**：是否使用了切角 (chamfer) 或 4px 小圆角？
- [ ] **字体**：按钮/徽章是否使用大写+加宽字距？
- [ ] **动效**：hover 是否有发光增强？active 是否有下压反馈？
- [ ] **阴影**：是否使用霓虹发光而非传统阴影？
- [ ] **边框**：是否使用发光边框？focus 时是否有明显的发光环？
- [ ] **差异化**：与默认主题对比，是否有明显的赛博朋克风格？

### 6.2 最终验收标准

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ✓ 通过标准：                                                                │
│                                                                             │
│  1. 截图对比：与默认主题截图对比，差异度明显                                   │
│  2. 盲测识别：给人看组件截图，能说出主题风格                                   │
│  3. 一致性：同类组件风格一致，切换主题时整体感觉统一                            │
│  4. 可用性：所有内容清晰可读，交互状态明确                                     │
│                                                                             │
│  ✗ 不通过：                                                                  │
│                                                                             │
│  1. 只有颜色不同，形状/动效/字体都与默认相同                                   │
│  2. 不同组件风格不统一                                                       │
│  3. 某些状态看不清或难以区分                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 7. 可复用样式代码片段

开发组件时，可直接使用以下代码片段：

### 7.1 切角 (Chamfer) 效果

```css
/* 小切角 - 用于按钮、徽章 */
.chamfer-sm {
  clip-path: polygon(
    6px 0, 100% 0, 100% calc(100% - 6px),
    calc(100% - 6px) 100%, 0 100%, 0 6px
  );
}

/* 中切角 - 用于卡片 */
.chamfer-md {
  clip-path: polygon(
    10px 0, 100% 0, 100% calc(100% - 10px),
    calc(100% - 10px) 100%, 0 100%, 0 10px
  );
}

/* 大切角 - 用于对话框 */
.chamfer-lg {
  clip-path: polygon(
    16px 0, 100% 0, 100% calc(100% - 16px),
    calc(100% - 16px) 100%, 0 100%, 0 16px
  );
}
```

### 7.2 霓虹发光效果

```css
/* 主色发光 - 按钮默认 */
.glow-primary {
  box-shadow:
    0 0 5px var(--primary),
    0 0 10px color-mix(in srgb, var(--primary) 50%, transparent),
    0 0 20px color-mix(in srgb, var(--primary) 30%, transparent);
}

/* 主色发光增强 - Hover */
.glow-primary-intense {
  box-shadow:
    0 0 8px var(--primary),
    0 0 16px var(--primary),
    0 0 32px color-mix(in srgb, var(--primary) 50%, transparent);
}

/* 次要色发光 */
.glow-secondary {
  box-shadow:
    0 0 5px var(--secondary),
    0 0 10px color-mix(in srgb, var(--secondary) 50%, transparent);
}

/* 内发光 - Active/Focus */
.glow-inset {
  box-shadow:
    inset 0 0 10px color-mix(in srgb, var(--primary) 30%, transparent),
    0 0 5px var(--primary);
}
```

### 7.3 边框发光

```css
/* 静态发光边框 */
.border-glow {
  border: 1px solid var(--primary);
  box-shadow:
    inset 0 0 8px color-mix(in srgb, var(--primary) 10%, transparent),
    0 0 4px color-mix(in srgb, var(--primary) 30%, transparent);
}

/* Focus 环 */
.focus-glow {
  outline: none;
  box-shadow:
    0 0 0 2px var(--background),
    0 0 0 4px var(--primary),
    0 0 12px var(--primary);
}
```

### 7.4 字体样式

```css
/* 按钮/徽章文字 */
.text-cyber {
  text-transform: uppercase;
  letter-spacing: 0.05em;
  font-weight: 500;
}

/* 可选：等宽风格 */
.text-mono {
  font-family: ui-monospace, "SF Mono", "Cascadia Mono", monospace;
}
```

### 7.5 过渡动画

```css
/* 标准过渡 */
.transition-cyber {
  transition: all 150ms ease-out;
}

/* Hover 上移 */
.hover-lift:hover {
  transform: translateY(-1px);
}

/* Active 下压 */
.active-press:active {
  transform: translateY(1px);
}
```
